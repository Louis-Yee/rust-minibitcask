# Mini Bitcask KV Engine
基于 Rust 从零实现的 **Bitcask 模型磁盘持久化 KV 存储引擎**，采用经典 Append-Only 追加写日志架构，深入实践文件 I/O、二进制编解码、内存索引、日志垃圾回收等底层存储技术。

## 项目简介
本项目参考经典 **Bitcask** 存储模型，使用纯 Rust 开发一款轻量级日志型 KV 存储引擎，具备**高写入性能、数据持久化、崩溃恢复、多进程安全**等特性。
核心设计思想：**磁盘仅追加写入 + 内存索引加速查询**，是学习数据库内核、文件系统 IO、Rust 系统编程的典型实战项目。

### 核心特性
- ✅ **Append-Only 追加写日志**：自定义二进制日志格式，所有写入仅在文件尾部追加，不修改历史数据，保证顺序写高性能
- ✅ **墓碑(Tombstone) 逻辑删除**：通过特殊标记实现软删除，无需物理擦除数据，适配日志型存储设计规范
- ✅ **崩溃自动恢复**：程序重启后回放全量日志，重建内存索引，异常宕机不丢失已落盘数据
- ✅ **BTreeMap 有序内存索引**：构建 KeyDir 索引，实现 O(logN) 单点查询；原生支持**范围扫描、前缀扫描**
- ✅ **日志合并(Merge) GC**：自动清理过期数据、删除标记，回收磁盘冗余空间
- ✅ **多进程安全**：基于 `fs4` 实现**进程级排他文件锁**，防止多进程并发读写冲突
- ✅ **数据持久化**：支持手动刷盘 + `Drop` 自动兜底落盘，保证操作系统缓存数据持久化到磁盘
- ✅ **完善单元测试**：覆盖读写、删除、重启恢复、遍历、日志合并等全核心场景

### 日志二进制格式
单条日志记录固定结构（大端字节序，跨平台兼容）：
+-------------+-------------+----------------+----------------+
| key_len (4B) | val_flag (4B) | key (变长) | value (变长) |
+-------------+-------------+----------------+----------------+
plaintext
1. `key_len`：4 字节，Key 字节长度（大端）
2. `val_flag`：4 字节，区分正常数据 / 墓碑标记
   - 正数：代表 Value 长度（正常 KV 数据）
   - `-1`：墓碑标记（逻辑删除）
3. `key`：变长字节流，原始 Key 数据
4. `value`：变长字节流，原始 Value 数据（删除时无此字段）

### 核心流程
1. **写入(Set/Delete)**：数据**仅追加**到日志文件尾部，同步更新/删除内存索引
2. **读取(Get)**：查询内存索引拿到磁盘偏移与长度，直接随机读文件，查询高效
3. **重启恢复**：从头遍历日志，逐条重建内存索引，识别墓碑完成状态同步
4. **日志合并(Merge)**：新建临时文件，仅重写有效 KV，原子重命名替换旧文件，清理垃圾数据
